# План тестирования и автоматизации мобильного приложения «Новости хосписа»

## 1. Описание приложения

Приложение предоставляет сотрудникам/волонтерам хосписа функционал для:

- Просмотра, фильтрации, сортировки и управления новостями (CRUD)
- Просмотра тематических цитат (в разделах «Главная» и «Цитаты»)
- Ознакомления с информацией о приложении, версией и правовыми документами

## 2. Определение границ и функционала приложения

### 2.1 Экран авторизации

- **Поля:** Логин, Пароль
- **Кнопка:** Войти

### 2.2 Основной экран (после авторизации)

- **Верхняя панель:** Навигационная кнопка «Гамбургер» (меню), Иконка «Бабочка» (Цитаты), Иконка «Авторизации» (Выйти)
- **Меню-гамбургер** (открывает): Главная, Новости, О приложении

### 2.3 Раздел «Главная»

- Графический элемент **«Аккордеон»** (свернутые/развернутые блоки с описанием)

### 2.4 Раздел «Новости» (основной список)

- Кнопка «Сортировка»
- Кнопка «Фильтрация» → переход на экран «Фильтровать новости»:
  - Выпадающий список «По категории» (8 категорий)
  - Два поля «Дата начала» и «Дата конца»
  - Кнопки Фильтрация и Отмена
- Кнопка «Создание новости» → переход в Контрольную панель

### 2.5 Контрольная панель (управление новостями)

- Кнопка «Сортировка»
- Кнопка «Фильтрация» (аналогично п.2.4, но доп. чекбоксы: Активная, Не активная)
- Кнопка «Создание новости» (форма создания):
  - Выпадающий список Категория (8 вариантов)
  - Поле Заголовок
  - Поле Дата публикации
  - Поле Время публикации
  - Поле Описание
  - Радиокнопка Активная / Не активная
  - Кнопки Сохранить и Отмена

### 2.6 Раздел «О приложении»

- Кнопка навигации Назад
- Номер версии приложения
- Ссылка Политика конфиденциальности
- Ссылка Пользовательское соглашение

### 2.7 Раздел «Цитаты» (доступ через иконку «Бабочка»)

- Графический элемент «Аккордеон» (развернуть для отображения текста цитаты)

## 3. Область тестирования (Checklist)

### 3.1 Функциональное тестирование

**Авторизация и выход:**

- Вход с валидными данными
- Вход с невалидными данными (пустые поля, неверный пароль, несуществующий логин)
- Выход из учетной записи (иконка → Выйти)

**Навигация:**

- Переходы через меню-гамбургер: Главная → Новости → О приложении
- Возврат на Главную из других разделов
- Открытие/закрытие меню-гамбургера

**Раздел «Новости» (основной список):**

- Сортировка новостей (по дате, по категории — согласно ТЗ)
- Фильтрация по категории и диапазону дат (проверка комбинаций)
- Сброс фильтров (кнопка Отмена)
- Переход в контрольную панель (кнопка Создание новости)

**Контрольная панель (CRUD новостей):**

- *Создание:*
  - С пустыми полями (валидация, кнопка Сохранить неактивна/ошибки)
  - С валидными значениями (все поля корректны)
  - С невалидными значениями (спецсимволы, длинный заголовок, дата в прошлом, некорректное время)
  - С активной/неактивной радиокнопкой
- *Сохранение:* Проверка появления новости на странице Главная и в списке Новости
- *Редактирование:* Изменение заголовка, даты, статуса активности
- *Удаление:* Удаление новости (подтверждение, исчезновение из списков)

**Дата/время публикации:**

- Установка прошедшей даты (должна отображаться, но возможно с пометкой)
- Установка текущей даты+времени
- Установка будущей даты (проверка, появляется ли в момент наступления)

**Раздел «О приложении»:**

- Отображение корректной версии (соответствие build.gradle)
- Переход по ссылке «Политика конфиденциальности» (открывается браузер/WebView)
- Переход по ссылке «Пользовательское соглашение»
- Кнопка Назад возвращает в предыдущий раздел

**Раздел «Цитаты» и «Главная» (аккордеон):**

- Разворачивание/сворачивание каждого элемента
- Одновременное раскрытие нескольких элементов
- Корректное отображение текста цитаты

### 3.2 Тестирование UI/UX

- Адаптивность под разные размеры экранов (эмуляторы Pixel, Nexus, планшеты)
- Отображение состояния загрузки (индикаторы)
- Поведение при повороте экрана (сохранение введенных данных, состояния аккордеона)
- Кликабельность элементов (минимальный размер 48dp)

### 3.3 Нефункциональное тестирование

- **Установка/запуск/удаление:** Чистая установка, обновление, удаление без ошибок
- **Оффлайн-режим:** Поведение при отсутствии интернета (кэш, ошибки API)
- **Производительность:** Скорость загрузки списка новостей при 100+ элементах, плавность скролла
- **Нагрузка:** Многократное создание/удаление новостей, фильтрация

### 3.4 Исследовательское тестирование (Ad-hoc)

- Отсутствие полной документации → поиск скрытых переходов, крашей при нестандартных действиях (быстрое нажатие кнопок, свайпы в неожиданных местах)

## 4. Автоматизированное тестирование

### 4.1 Выбор инструментов

- **Фреймворк:** Espresso (UI) + JUnit4/5
- **Язык:** Kotlin
- **Сборка:** Gradle
- **Отчеты:** Allure + плагин для Android
- **Прокси/сеть:** Charles Proxy (для подмены ответов API)
- **Эмулятор:** Android API 29

### 4.2 Стратегия автоматизации (Pyramid)

- **Уровень 1: Модульные тесты (Unit) — 20%**
  - Тестирование валидаторов полей (заголовок, дата, время)
  - Тестирование логики фильтрации на стороне приложения
  - Тестирование парсеров JSON (ответов API)

- **Уровень 2: Интеграционные тесты — 30%**
  - Взаимодействие с локальной БД (Room) — создание/удаление новостей
  - Работа с SharedPreferences (сохранение сессии, настроек фильтров)

- **Уровень 3: UI-тесты (Espresso) — 50%**
  - Авторизация: 3 сценария (успех, неверный пароль, пустые поля)
  - Навигация: проверка открытия всех разделов из гамбургера
  - Новости: сортировка, фильтрация + сброс, создание/редактирование/удаление новости, валидация полей
  - Аккордеон (Цитаты/Главная): проверка раскрытия/скрытия и текста
  - О приложении: проверка версии, Intent-тест на открытие браузера

### 4.3 Пример теста для Espresso (Kotlin)

```kotlin
@Test
fun createNews_validData_success() {
    onView(withId(R.id.button_create_news)).perform(click())
    onView(withId(R.id.category_spinner)).perform(click())
    onView(withText("Праздник")).perform(click())

    onView(withId(R.id.description_input)).perform(typeText("Поздравляем всех!"))
    onView(withId(R.id.save_button)).perform(click())
    onView(withText("День медсестры")).check(matches(isDisplayed()))
}
4.4 Работа с нестабильностью (Flakiness)
Использовать IdlingResource для ожидания загрузки данных из API

Для тестов с датой/временем — внедрение зависимостей (подменить Clock на фиксированную дату)

Для фильтрации — подготовить тестовые данные через API (clean state перед тестом)

4.5 CI/CD (рекомендация)
Запуск автотестов на каждый Pull Request

Ночные прогоны полного набора

Интеграция с Allure для просмотра истории прогонов

6. Критерии качества (Definition of Done для тестирования)
Функциональные тесты (ручные) — 100% покрытие чеклиста из п.3

Автотесты — проходят на эмуляторе и реальном устройстве без ошибок (стабильность > 98%)

Отчет Allure — содержит скриншоты упавших тестов и логи

Исследовательское тестирование — не выявило критических багов (crash, data loss)

UI-тесты — покрывают минимум: авторизацию, CRUD новостей, фильтрацию, аккордеон
